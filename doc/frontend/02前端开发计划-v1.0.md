# 视频编辑器前端开发计划 v1.0

## 1. 文档目的

本文档用于整理当前阶段的前端执行计划。

范围约束：

- 本阶段只做前端开发
- 需求文档仍可继续补充前后端内容
- 后端能力暂以接口约束、mock 数据和占位适配层为准

## 2. 当前前提

当前前端开发以以下约束为准：

- 前端框架使用 `Next.js 16`
- 视频编辑相关能力统一使用 `Remotion`
- 前端全局状态优先使用 `zustand`
- 基础组件统一使用 `shadcn/ui` 与 `Radix`
- 基础 UI 组件统一放在 `components/ui`
- 业务代码按 `features` 拆分
- 页面组件尽量不直接承担请求、转换和状态编排逻辑
- 一个项目支持多个视频片段和多个音轨片段
- 导入视频时默认拆分视频轨和音频轨
- 未明确的视频编辑交互默认参考 `Final Cut Pro`
- 模型页当前只按 `API 地址 + key + 帮助文档` 设计
- 暂不考虑应用内模型下载

## 3. 本阶段目标

前端第一阶段的目标不是把所有 AI 能力跑通，而是先把一个可持续迭代的编辑器前端骨架做稳。

本阶段应达成：

- 可进入应用并完成项目创建 / 打开流程
- 可进入编辑器主界面
- 可展示多视频片段、多音轨片段的时间轴基础结构
- 可在 `Remotion` 中完成预览、选中、切片表达
- 可展示素材、分析结果、音色、TTS、模型配置、导出等主要页面骨架
- 通过 mock 数据先跑通关键交互链路

本阶段不追求：

- 真正完成后端任务调度
- 真正完成模型调用和导出执行
- 在第一轮就把 `project.json` 字段完全定死

## 4. 开发原则

### 4.1 状态分层

- `zustand` 负责跨区域共享状态
- 组件内部状态只保留局部交互状态
- 与项目、时间轴、播放头、选区、导出配置相关的状态进入 store
- 与弹窗开关、输入临时值相关的状态保留在组件局部

### 4.2 结构分层

- `components/ui` 只放基础组件
- `components` 放可复用的通用展示组件
- `features` 放业务模块
- `stores` 放状态仓库
- `lib` 或 `services` 放数据适配和接口封装

### 4.3 数据策略

- 所有页面先基于稳定的前端视图模型开发
- 不直接依赖最终版 `project.json`
- 为项目、时间轴、素材、TTS、模型配置分别建立前端适配对象
- 后续再把适配对象映射到真实后端协议

### 4.4 视频编辑策略

- 时间轴、片段、选区、播放头、预览、导出视图统一围绕 `Remotion`
- 不并行维护另一套视频编辑表达
- 先做单一表达模型，再逐步接入 AI 结果和导出能力

## 5. 分阶段计划

### 5.1 第一阶段：工程基线与目录骨架

目标：

- 初始化前端工程结构
- 固化技术栈和目录约束

交付内容：

- `Next.js 16` 基础工程
- `Remotion` 集成骨架
- `zustand` store 基础目录
- `shadcn/ui` 初始化
- `components/ui`、`features`、`stores`、`lib`、`types` 目录
- 全局主题、布局和路由骨架

完成标准：

- 项目可启动
- 页面路由可访问
- 基础 UI 组件体系和目录约束已固定

### 5.2 第二阶段：项目入口与应用壳

目标：

- 完成应用启动后的项目流

交付内容：

- 启动页
- 新建项目入口
- 打开项目文件夹入口
- 最近项目列表占位
- 项目基础信息展示

完成标准：

- 用户可以从入口页进入编辑器
- 项目会话状态已进入 store

### 5.3 第三阶段：编辑器主界面骨架

目标：

- 先把主界面四区结构稳定下来

交付内容：

- 左侧素材 / 分析 / 模型导航区域
- 中部预览区域
- 下方时间轴区域
- 右侧属性与操作面板
- 顶部项目与导出入口

完成标准：

- 编辑器主界面可正常切换主要模块
- 布局在桌面端下稳定可用

### 5.4 第四阶段：时间轴数据模型与 Remotion 预览

目标：

- 建立前端第一版时间轴表达模型

交付内容：

- 多视频片段结构
- 多音轨片段结构
- 播放头状态
- 选中片段状态
- 时间轴缩放状态
- `Remotion` 预览联动

完成标准：

- 时间轴与预览之间可联动
- 可展示多个视频片段和多个音轨片段
- 可选中片段并反映到属性区

### 5.5 第五阶段：素材导入与媒体面板

目标：

- 完成素材面板与导入后视图表达

交付内容：

- 视频素材列表
- 音频素材列表
- 导入状态展示
- 导入后视频轨 / 音频轨分离结果展示
- 素材预览与基础信息展示

完成标准：

- 导入流程的前端交互跑通
- 导入结果可映射到时间轴视图模型

### 5.6 第六阶段：分析结果展示与智能剪辑入口

目标：

- 先把分析结果消费界面做好

交付内容：

- 视频分析结果面板
- 字幕 / transcript 面板
- 关键帧缩略图区
- 剪辑工具入口
- 候选结果列表

完成标准：

- 分析结果可视化完整
- 用户可从结果面板进入剪辑操作入口

说明：

- `VL -> 剪辑动作` 的自动映射当前先不做闭环
- 第一版先展示结果和候选入口

### 5.7 第七阶段：语音、音色与 TTS 模块

目标：

- 跑通音频插入类交互

交付内容：

- 录音入口 UI
- 音频导入 UI
- 音色库界面
- 上传样本创建音色界面
- 录音创建音色界面
- 从项目片段选择音色样本界面
- TTS 文本输入与参数表单
- TTS 试听和生成结果区域
- 生成后插入时间轴的交互

完成标准：

- 用户可以完成“选择样本 -> 输入文字 -> 生成片段 -> 插入音轨”的前端流程

### 5.8 第八阶段：模型页与依赖页

目标：

- 完成设置侧的关键页面

交付内容：

- VL / LLM / 音频处理 / TTS 四类模型配置页
- API 地址与 key 配置表单
- 帮助文档入口
- 依赖检测页
- 依赖状态、日志和安装反馈区域

完成标准：

- 模型页可承载当前 API 接入方案
- 依赖页可承载本地环境检查交互

### 5.9 第九阶段：导出页

目标：

- 先完成导出配置与任务视图

交付内容：

- 导出参数面板
- 导出预设列表
- 导出目录选择
- 导出任务状态列表

完成标准：

- 导出页交互完整
- 与时间轴和项目状态具备前端联动关系

### 5.10 第十阶段：联调准备与结构收口

目标：

- 为后续真实后端接入做准备

交付内容：

- mock 数据整理
- store 清理
- 业务组件与基础组件边界收口
- 适配层接口整理
- 页面级状态流梳理

完成标准：

- 前端可切换到真实接口而不大幅返工
- 时间轴、素材、模型、TTS、导出几条主链路的数据入口统一

## 6. 建议的优先级

建议按以下顺序执行：

1. 第一阶段到第四阶段
2. 第五阶段到第七阶段
3. 第八阶段到第十阶段

原因：

- 先把编辑器骨架和 `Remotion` 数据表达做稳
- 再补素材、分析结果和 TTS 这些高频模块
- 最后收口设置页、导出页和联调适配层

## 7. 当前仍会影响前端设计的关键问题

这些问题不必现在全部回答，但需要在开发过程中持续收敛：

- `project.json` 的最终字段结构
- 多视频片段与多音轨的最终时间轴数据结构
- “从现有片段创建音色”是生成全局音色，还是仅本次引用
- 模型页是否只保留配置与文档，还是增加最小连通性检查
- 导入模式与引用模式下素材路径如何在前端展示

## 8. 本计划的执行边界

本计划是前端开发计划，不是完整研发排期。

因此它只负责回答三件事：

- 前端先做什么
- 前端怎样拆模块
- 前端如何降低后续返工风险

后续如果你开始按版本推进，可以继续在此基础上扩展：

- `v1.1` 前端调整计划
- `v1.2` 前端调整计划
- `v2.0` 下一阶段计划
