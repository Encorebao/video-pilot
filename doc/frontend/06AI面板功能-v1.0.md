# AI 面板功能文档 v1.0

> 对应左侧侧边栏 **AI** Tab（原"分析"）

---

## 一、面板概览

AI 面板提供两类能力：**AI Skills（一键自动化操作）** 和 **AI 聊天对话**。  
技能区在上、对话区在中、输入栏固定在底部。

---

## 二、AI Skills

### 2.1 定义

Skills 是预置的 AI 自动化剪辑动作，每个 Skill 包含：

| 字段 | 说明 |
|---|---|
| `id` | 唯一标识符（slug 格式，如 `remove-silence`） |
| `label` | 展示名称 |
| `desc` | 一句话描述 |
| `prompt` | （未来）存储在数据库中的 Markdown 提示词 |

### 2.2 当前内置 Skills

| ID | 名称 | 描述 |
|---|---|---|
| `remove-silence` | 剪掉空白片段 | 自动检测并删除超过 0.5s 的静音段 |
| `remove-filler` | 剪掉口头禅 | 识别并剪除「那个」「就是」「嗯」等口头禅 |
| `auto-broll` | 智能 B-roll 插入 | 根据旁白语义推荐并插入 B-roll 素材 |
| `color-grade` | 一键调色 | 分析画面风格，自动应用 LUT 调色方案 |
| `highlight-reel` | 精华集锦 | 提取高信息密度片段生成精华剪辑 |

### 2.3 交互流程

1. 用户点击 Skill 卡片按钮
2. 按钮进入 loading 态（`Loader2` 旋转 + 紫色高亮）
3. 执行完成后，在聊天区输出结果消息
4. 当有 Skill 在执行时，其余 Skill 按钮禁用（一次只能执行一个）

### 2.4 数据库存储（TODO — 未实现）

Skills 将以 **Markdown 文件**的形式存储在数据库，结构规划如下：

```markdown
---
id: remove-silence
label: 剪掉空白片段
version: 1
author: system
tags: [audio, edit]
---

## 描述
自动检测视频/音频中的静音段，并将超过阈值的部分从时间轴中删除。

## 参数
- threshold: 0.5s（静音判定阈值）
- min_voice_gap: 0.1s（保留的最小语音间隔）

## 执行逻辑
1. 调用后端 `/api/analyze/silence` 接口，传入媒体 ID
2. 接收返回的静音时间段列表
3. 对每个时间段调用 `splitClip` + `deleteClip` 操作
```

未来支持：
- 用户自定义 Skill（写 Markdown 即可）
- Skill 市场（社区共享）
- Skill 参数化（调整阈值等）

## 三、AI 聊天

### 3.1 消息格式

| 角色 | 气泡位置 | 样式 |
|---|---|---|
| `user` | 右对齐 | `bg-white/[0.1]` |
| `assistant` | 左对齐 | `bg-white/[0.05]` |

### 3.2 输入框

- 多行 `<textarea>`，最大高度 96px，超出滚动
- **Enter** 发送，**Shift+Enter** 换行
- 右侧发送按钮（↑ 箭头图标），内容为空时禁用

### 3.3 当前实现（Mock）

- 用户发送消息后，800ms 后返回固定的 Mock 回复
- Skill 执行结果也以消息形式输出到聊天区
- 后续对接：替换 `sendMessage()` 内的 setTimeout 为真实 API 调用

## 四、相关 Store

| Store | 字段/方法 |
|---|---|
| `ui-store` | `analyzingIds`、`analyzedIds`、`analyzeItem()` |

AI 聊天状态（messages、input）目前为组件 local state，如需持久化/跨面板共享可迁移到 `ui-store`。
